## 1. 设计概述

### 1.1 存储策略

| 数据类型 | 存储方式 | 理由 |
|----------|----------|------|
| **配置数据** | JSON 文件 | 人类可读、易于编辑和同步 |
| **Fence 布局** | JSON 文件 | 频繁读写、结构化数据 |
| **规则定义** | JSON 文件 | 易于导入导出 |
| **主题配置** | JSON 文件 | 独立文件便于分享 |
| **图标缓存** | SQLite | 大量二进制数据、需要索引 |
| **使用统计** | SQLite | 聚合查询、数据分析 |

### 1.2 文件存储位置

```
%APPDATA%\Fences6\
├── config.json              # 全局配置
├── fences.json              # Fence 布局 (JSON)
├── rules.json               # 自动化规则 (JSON)
├── portal_cache.json        # Portal 文件索引
├── themes\                  # 主题目录
│   ├── dark.json
│   └── custom-themes\
├── backups\                 # 自动备份
├── logs\                    # 日志
└── cache\
    └── icon_cache.db        # SQLite 图标缓存
```

## 3. SQLite 数据库设计

### 3.1 icon_cache.db 结构

用于缓存图标和文件元数据，提升性能。

```sql
-- 图标缓存表
CREATE TABLE icon_cache (
    file_path TEXT PRIMARY KEY,
    icon_data BLOB NOT NULL,
    icon_width INTEGER NOT NULL,
    icon_height INTEGER NOT NULL,
    cached_date TEXT NOT NULL,
    file_hash TEXT NOT NULL,
    source_type TEXT DEFAULT 'system'  -- 'system', 'custom', 'portal'
);

-- Portal 文件索引表
CREATE TABLE portal_index (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fence_id TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_name TEXT NOT NULL,
    file_ext TEXT,
    file_size INTEGER,
    modified_date TEXT,
    is_directory INTEGER DEFAULT 0,
    indexed_date TEXT NOT NULL,
    UNIQUE(fence_id, file_path)
);

CREATE INDEX idx_portal_fence ON portal_index(fence_id);
CREATE INDEX idx_portal_name ON portal_index(file_name);
CREATE INDEX idx_portal_ext ON portal_index(file_ext);

-- 使用统计表
CREATE TABLE usage_stats (
    item_path TEXT PRIMARY KEY,
    use_count INTEGER DEFAULT 0,
    last_used TEXT,
    created_date TEXT,
    category TEXT  -- 'app', 'document', 'folder', 'game'
);

CREATE INDEX idx_usage_count ON usage_stats(use_count DESC);
CREATE INDEX idx_usage_last ON usage_stats(last_used DESC);
```

## 5. 数据同步

### 5.1 自动备份策略

| 事件 | 触发条件 | 保留数量 |
|------|----------|---------|
| **定时备份** | 每 24 小时 | 7 个 |
| **启动前备份** | 应用启动 | 3 个 |
| **修改前备份** | 每次保存前 | 1 个 |
| **手动备份** | 用户触发 | 无限 |

### 5.2 备份文件格式

```
backups/
├── 2026-05-02_143022_config.json    # 配置备份
├── 2026-05-02_143022_fences.json    # Fence 布局备份
├── 2026-05-02_100015_config.json    # 早期备份
└── 2026-05-01_080000_config.json    # 更早备份
```

## 6. 性能考虑

### 6.1 大数据量优化

| 场景 | 优化策略 |
|------|---------|
| **大量图标** | 虚拟化渲染，只加载可视区域图标 |
| **大型文件夹 Portal** | 分页加载，懒加载子目录 |
| **频繁文件变更** | 节流/防抖，避免频繁刷新 |
| **图标缓存** | LRU 缓存，限制缓存大小 |

### 6.2 索引策略

```sql
-- Portal 索引优化
CREATE INDEX idx_portal_fence_modified 
ON portal_index(fence_id, modified_date DESC);

-- 使用统计查询优化
CREATE INDEX idx_usage_recent 
ON usage_stats(last_used DESC) WHERE use_count > 0;
```
