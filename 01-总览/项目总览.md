## 1. 项目概述

### 1.1 产品定义

Fences 6 是一款 **Windows 桌面图标与文件组织管理工具**，通过在桌面上创建可自定义的半透明"围栏"（Fence）区域，将快捷方式、文件、文件夹智能分类整理，解决桌面图标杂乱、查找低效的核心痛点。

### 1.2 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **零学习成本** | 不替代桌面，只是在桌面上叠加组织层 |
| **自动化优先** | 规则驱动的自动分类，一次配置永久生效 |
| **视觉个性化** | 高度可定制的外观，融入用户审美 |
| **效率提升** | Peek、Portals 等功能让桌面成为真正的生产力中心 |

### 1.3 目标用户

| 用户群 | 典型场景 |
|--------|---------|
| **日常用户** | 桌面图标过多，需要快速找到应用和文件 |
| **游戏玩家** | Steam、Epic 等多平台游戏图标堆积 |
| **学生/知识工作者** | 按课程/项目分类管理文档 |
| **企业/IT 管理员** | 批量部署、统一桌面配置管理 |
| **设计师/创作者** | 需要整洁桌面同时快速访问素材 |

### 1.4 版本规划

| 版本 | 功能范围 | 定价策略 |
|------|---------|---------|
| **Free** | 基础 Fence + 3 个分区上限 | 免费 |
| **Pro** | 无限 Fence + Portals + Peek + 自动化 + 主题 | 订阅制 / 买断 |
| **Enterprise** | Pro 全功能 + 部署工具 + 配置锁定 + 优先支持 | 按设备授权 |

### 3.2 技术选型

| 层级 | 技术方案 | 理由 |
|------|---------|------|
| **UI 框架** | WPF / WinUI 3 | 原生 Windows 渲染，支持亚克力/Acrylic 材质 |
| **核心引擎** | C++ / Rust | 高性能文件监控和布局计算 |
| **配置存储** | JSON + SQLite | JSON 存布局配置，SQLite 存规则和索引 |
| **文件监控** | `ReadDirectoryChangesW` | Win32 原生 API，低开销实时文件变更通知 |
| **Shell 集成** | COM Shell Extension | 拦截桌面图标创建/删除事件 |
| **热键系统** | `RegisterHotKey` / Low-level Keyboard Hook | 全局热键注册 |
| **安装包** | MSIX / WiX | 支持静默安装和企业部署 |

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Windows 更新破坏 Shell 集成 | 核心功能不可用 | 采用多方案兼容（Shell Extension + Subclass + Overlay） |
| 杀毒软件误报 | 用户安装受阻 | 提前提交白名单申请，代码签名证书 |
| 高 DPI 渲染异常 | Fence 位置/大小错乱 | Per-Monitor V2 + 全面 DPI 测试矩阵 |
| 大量图标性能问题 | 卡顿、内存溢出 | 虚拟化渲染 + 懒加载 + 性能预算 |
| 与同类产品冲突 | 图标管理冲突 | 检测并提示关闭竞品，协商共存方案 |
